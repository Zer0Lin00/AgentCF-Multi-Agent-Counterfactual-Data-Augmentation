# AgentCF 实验执行文档

## 1. 项目名称

**AgentCF: Multi-Agent Counterfactual Data Augmentation for Robust Text Classification**

---

## 2. 项目目标

实现一个用于文本分类的数据增强框架 **AgentCF**。  
该框架通过多个 agent 协作生成高质量反事实样本（counterfactual samples），并将这些样本加入训练集，以提升模型在以下方面的表现：

1. 标准测试集性能
2. 低资源场景表现
3. OOD / challenge set 鲁棒性
4. 对 shortcut / spurious correlation 的抵抗能力

---

## 3. 核心研究问题

### RQ1

多 agent 协作是否能生成比单一 LLM 更高质量的反事实样本？

### RQ2

用多 agent 生成的反事实样本做数据增强，是否能提升分类模型性能与鲁棒性？

### RQ3

在多 agent 框架中，Planner、Verifier、Selector 哪些模块最关键？

---

## 4. 总体方法概述

AgentCF 包含 4 个 agent：

1. **Planner Agent**
   - 输入原始样本和目标标签
   - 规划哪些内容应该改，哪些内容必须保持不变

2. **Generator Agent**
   - 根据 Planner 输出生成多个候选反事实样本

3. **Verifier Agent**
   - 评估候选样本是否满足：
     - 目标标签正确
     - 语义保持
     - 改动最小
     - 干预一致

4. **Selector Agent**
   - 根据评分筛选高质量反事实样本
   - 只保留满足阈值的样本进入增强训练集

此外，引入闭环反馈机制（Refinement Loop）：

- 当 Verifier 判定候选不合格时，必须输出可执行失败原因（例如“情感未翻转”“改动了核心实体”“语义偏离过大”）。
- Generator 根据失败原因重写候选，最多重试 N 轮（建议 `max_retry_rounds = 2`）。
- 若达到重试上限仍不合格，再丢弃该样本。

要求：

- 电影主题、讨论对象尽量不变
- 只改情感相关表达
- 尽量最小修改

完整的多 agent 反事实生成与筛选流程。

## 9. AgentCF 详细实现

### 9.1 Planner Agent
输入字段：

```json
{
  "id": "sample_0001",
  "text": "The movie is wonderful and touching.",
  "label": 1,
  "target_label": 0
}
```

输出字段（Plan Schema）：

```json
{
  "id": "sample_0001",
  "target_label": 0,
  "causal_features": ["movie entity", "main event"],
  "spurious_features": ["sentiment adjectives", "emotional tone"],
  "elements_to_change": ["wonderful", "touching"],
  "elements_to_preserve": ["movie", "event structure"],
  "editing_constraints": [
    "minimal edits",
    "do not change named entities unless necessary",
    "keep fluent and natural"
  ]
}
```

执行约束：

- 只输出结构化规划，不直接改写文本。
- 必须显式区分 `causal_features` 与 `spurious_features`。
- 优先干预伪相关线索，保留因果相关核心事实。

### 9.2 Generator Agent
输入字段：

```json
{
  "id": "sample_0001",
  "text": "The movie is wonderful and touching.",
  "plan": {},
  "feedback": {
    "round": 1,
    "critique": "情感翻转不足，且改动了实体名"
  }
}
```

输出字段（Candidates Schema）：

```json
{
  "id": "sample_0001",
  "target_label": 0,
  "round": 1,
  "candidates": [
    {"candidate_id": "c1", "text": "The movie is dull and emotionally flat."},
    {"candidate_id": "c2", "text": "The movie feels lifeless and uninspired."},
    {"candidate_id": "c3", "text": "The movie is painfully bland and cold."}
  ]
}
```

执行约束：

- 默认每条样本生成 3 个候选，可扩展到 5 个。
- 仅修改与标签相关关键部分，保持原语义框架与实体一致。
- 若 Verifier 返回 `critique`，必须按反馈重写。
- 达到 `max_retry_rounds` 后仍不达标则丢弃样本。

### 9.3 Verifier Agent

输入字段：

```json
{
  "id": "sample_0001",
  "original_text": "The movie is wonderful and touching.",
  "original_label": 1,
  "target_label": 0,
  "plan": {},
  "candidate": {"candidate_id": "c1", "text": "The movie is dull and emotionally flat."}
}
```

评分维度：

1. `label_score`：目标标签正确性（优先独立判别器，可加 LLM 二审）。
2. `semantic_score`：语义保持（embedding/BERTScore + 去情感词 overlap）。
3. `minimality_score`：最小改动（edit distance、token change ratio）。
4. `consistency_score`：与 Planner 干预约束一致性。

输出字段（Verification Schema）：

```json
{
  "id": "sample_0001",
  "candidate_id": "c1",
  "label_score": 0.92,
  "semantic_score": 0.88,
  "minimality_score": 0.81,
  "consistency_score": 0.85,
  "final_score": 0.87,
  "status": "pass",
  "critique": ""
}
```

综合评分公式：

```text
final_score =
0.35 * label_score +
0.30 * semantic_score +
0.20 * minimality_score +
0.15 * consistency_score
```

### 9.4 Selector Agent
输入：Verifier 的评分结果列表。

筛选逻辑：

- 硬阈值过滤：`label_score >= 0.80`，`semantic_score >= 0.80`，`final_score >= 0.82`。
- 动态过滤（推荐）：按文本长度分桶，在每个桶内保留前 `percentile_keep`（如 30%）。
- 每个原始样本默认保留 `top_k=1`，可扩展为 `top_k=2`。

输出字段（Selected Schema）：

```json
{
  "original_id": "sample_0001",
  "original_label": 1,
  "selected_counterfactuals": [
    {
      "candidate_id": "c1",
      "text": "The movie is dull and emotionally flat.",
      "label": 0,
      "final_score": 0.89
    }
  ]
}
```

## 12. 评测指标

### 12.1 任务性能指标
Accuracy
Macro-F1

### 12.2 反事实质量自动指标
Label Flip Success Rate
Semantic Similarity
Edit Distance
Diversity Score
指标说明
Label Flip Success Rate

被目标标签分类器正确识别为目标标签的比例。
建议使用独立判别器或 LLM 二审进行核验。

Semantic Similarity

原始文本与反事实文本 embedding 的 cosine similarity。
可补充去情感词后的 overlap 指标，降低“仅改情感词”场景下的误判。

Edit Distance

原始文本与反事实文本之间的编辑距离。

Diversity Score

同一原始样本多个候选之间的平均差异。

### 12.3 鲁棒性指标

默认 OOD 设置：

- 训练域：SST-2
- OOD 域：IMDb（可扩展 Yelp Polarity / Amazon Reviews）

评测时记录：

In-domain Accuracy
OOD Accuracy
Robustness Gap = In-domain Acc - OOD Acc

若当期资源有限，可先完成 SST-2 -> IMDb，后续再扩展更多 OOD 域。

### 12.4 人工评估

随机抽样 50 到 100 个生成样本，请人工打分，维度如下：

Validity：是否符合目标标签
Minimality：是否改动尽量小
Fluency：是否自然流畅
Semantic Preservation：是否保留主要语义

评分建议：

1 到 5 分

## 13. 消融实验

必须实现以下消融：

### 13.1 w/o Planner

不做显式干预规划，直接生成反事实。

目的：

验证 Planner 是否有帮助

### 13.2 w/o Verifier Feedback

保留 Verifier 打分，但移除 Verifier -> Generator 的反馈重试闭环。

目的：

验证闭环反馈是否关键

### 13.3 w/o Selector

做 Verifier 打分，但不筛选，只全部保留。

目的：

验证筛选步骤的重要性

### 13.4 Single-Agent Version

把 Planner + Generator + Verifier 的功能压缩成一次 prompt。

目的：

和完整多 agent 框架比较

### 13.5 w/o Dynamic Threshold

仅使用固定阈值筛选，不做长度分桶/分位数筛选。

目的：

验证动态阈值对分布保持与效果的贡献

## 19. 关键 Prompt 设计要求

需要实现以下 prompt 模板：

### 19.1 Planner Prompt

功能：输入原句和目标标签，输出结构化干预计划。

模板：

```text
你是 Planner Agent。请只输出 JSON，不要输出解释。
任务：给定原句、原标签、目标标签，生成可执行干预计划。
要求：
1) 显式区分 causal_features 与 spurious_features
2) 指定 elements_to_change 与 elements_to_preserve
3) 给出 editing_constraints，强调 minimal edits

输入：
id={id}
text={text}
label={label}
target_label={target_label}
```

### 19.2 Generator Prompt

功能：输入原句 + Planner 输出 +（可选）Verifier critique，生成 3 到 5 个候选。

模板：

```text
你是 Generator Agent。请只输出 JSON，不要输出解释。
任务：根据 plan 生成反事实候选，目标标签必须为 {target_label}。
要求：
1) 保留因果相关核心事实与实体
2) 仅修改必要的情感/风格相关片段
3) 候选之间保持多样性
4) 若提供 critique，必须优先修复 critique 指出的失败点

输入：
id={id}
text={text}
plan={plan_json}
feedback={critique_or_empty}
num_candidates={num_candidates}
```

### 19.3 Verifier Prompt（可选 LLM 版本）

功能：输入原句、干预计划、候选文本，输出评分与失败原因。

模板：

```text
你是 Verifier Agent。请只输出 JSON，不要输出解释。
请对候选文本给出：
label_score, semantic_score, minimality_score, consistency_score, final_score, status, critique

评分范围：[0,1]
status 只能是 pass 或 reject
若 status=reject，critique 必须给出可执行修改建议

输入：
id={id}
original_text={original_text}
original_label={original_label}
target_label={target_label}
plan={plan_json}
candidate={candidate_text}
```

### 19.4 Single-Agent Baseline Prompt

功能：不区分 agent，一次性生成反事实文本（用于 baseline）。

模板：

```text
请将下面句子改写为目标标签文本，并尽量保持语义不变：
text={text}
label={label}
target_label={target_label}
要求：仅做最小必要改动，保留实体与主要事件。
```

## 20. 推荐默认阈值

初始默认值如下：

label_score >= 0.80
semantic_score >= 0.80
minimality_score >= 0.70
final_score >= 0.82

当使用动态筛选时，补充规则：

- 先按硬阈值过滤，再按长度分桶做分位数筛选
